# Goal 执行日志：文章发布小应用（2026-07-19）

## 目标

把已经验证的文章导入、检查、预览和发布命令封装成可以双击使用的 Windows 小应用，并使用两篇独立测试文章完成端到端验收和对抗式审查。

## 边界

- 不修改 7 篇正式归档文章。
- 测试文章只用于本地验收，不推送或部署线上。
- GUI 复用现有文章工具，不维护第二套发布规则。
- 不引入新的第三方依赖。

## 验收标准

- [x] 双击启动器可以打开文章发布小应用。
- [x] 用户无需输入命令即可选择原稿、导入、检查、预览和发布。
- [x] 第一篇测试文章通过工具发布并通过完整构建与搜索验证。
- [x] 第二篇独立测试文章重复完整流程仍然通过。
- [x] 对抗式审查未发现会覆盖文章、泄漏草稿、残留测试文章或后台服务的问题。
- [x] 两篇测试文章均被清理，最终网站内容不变。

## 验收记录

### 小应用与启动器

- 根目录 `启动文章发布工具.bat` 可正常启动 Windows 窗口，窗口标题为 `ChenTingToDo 文章发布工具`，进程响应正常。
- PowerShell 脚本保持 UTF-8 BOM；Windows PowerShell 5.1 语法解析错误数为 0。
- 界面封装了选择原稿、导入草稿、检查、浏览器预览、发布、打开 Markdown 和打开目录等操作。
- 发布器调用同一套 `article-tool.ts` 校验和发布逻辑，没有复制第二套规则。

### 两篇文章端到端测试

- 第一篇测试文章 `gui-wrapper-test-one`：通过小应用的同一执行入口完成导入与发布，完整 `npm run check` 通过。
- 第二篇独立测试文章 `gui-wrapper-test-two`：再次完成导入与发布，完整 `npm run check` 通过。
- 浏览器实测两篇正式文章页均可打开，首页文章列表可见，搜索“第二次验收”可命中第二篇文章，浏览器控制台无报错或警告。

### 对抗式审查

- 重复使用已存在的 slug 会失败退出；原文章 SHA-256 前后相同，证明不会静默覆盖。
- 使用一篇不带 frontmatter 的真实归档原稿执行导入检查成功；摘要和标签可以直接在界面填写，不要求用户手改 YAML 才能开始。
- 单元测试覆盖危险 HTML、疑似密钥、无效元数据、断链、重复 slug、发布失败回滚和中文空格路径，共 6 项测试全部通过。
- 发布前会执行 lint、类型检查、测试、正式内容检查、静态构建和 Pagefind 索引；失败时恢复草稿并删除不完整输出。
- 关闭应用会清理它启动的预览服务；验收结束后 4174–4176 端口均无监听，发布器窗口进程无残留。
