# ChenTingToDo 文章发布说明

## 推荐方式：双击小应用

直接双击项目根目录的：

```text
启动文章发布工具.bat
```

打开后按界面上的顺序操作：

1. 选择 Markdown 原稿；工具会在本机读取标题和开篇内容，自动推荐文章类型、一句话摘要和标签；
2. 查看推荐结果。可以直接使用，也可以按自己的表达修改；即使摘要和标签留空，导入时也会自动补全；
3. 点击“导入为草稿”；
4. 点击“检查草稿”。旧草稿缺少摘要或标签时，工具会先自动补全；其他问题会显示问题位置和修改建议；
5. 如需修改正文，点击“打开当前 Markdown”；
6. 点击“浏览器预览”；
7. 点击“正式上线”，确认公开后等待工具完成最终检查、GitHub 推送和 Vercel 生产部署。

应用关闭时会自动停止它启动的预览服务。“正式上线”会只提交当前文章并推送到 GitHub 的生产分支，随后等待 Vercel 完成生产部署。成功后可以直接打开线上文章。按钮执行前会再次确认，因为完成后互联网访客即可看到文章。

推荐摘要和标签使用本地规则生成，不调用付费 AI，也不会把原稿上传到第三方。标签优先识别标题和开篇中的产品名、技术名与文章主题，最多生成 5 个。若同一篇文章已经导入过，工具会引导你点击“继续已有草稿”，不会创建或覆盖重复文章。

## 命令行方式（高级用法）

1. 准备一个 Markdown 原稿。原稿可以在任意目录，文件名可以包含中文和空格。
2. 导入为草稿：

   ```powershell
   npm run article:prepare -- "G:\路径\原稿.md" --type troubleshooting
   ```

   `--type` 可选值：`troubleshooting`（故障复盘）、`learning`（学习记录）、`project`（项目记录）。如自动生成的英文 slug 不理想，可增加 `--slug my-article`。

3. 按命令输出的位置编辑 `content\drafts\slug.md`，把 `TODO`、标签和分类补充完整。
4. 检查文章：

   ```powershell
   npm run article:check -- slug
   ```

5. 使用正式文章页样式预览草稿：

   ```powershell
   npm run article:preview -- slug
   ```

   浏览器打开命令显示的地址。草稿只在该本地预览进程中可见，不会进入正式文章列表。

6. 生成已验收的正式文章：

   ```powershell
   npm run article:publish -- slug
   ```

   该命令会先校验文章，然后临时移入正式目录并执行 lint、类型检查、测试、静态构建和 Pagefind 索引生成。任何一步失败都会恢复原草稿。

7. 检查 `git diff`。确认内容无误后，再自行提交和推送；工具不会自动操作 Git 或部署线上。

## 安全规则

- 新导入文章始终是 `published: false`，只有点击“正式上线”并确认后才会公开。
- 正式构建只收录明确写有 `published: true` 的文章。
- 不要在文章中粘贴真实 API Key、Token、私钥、账号或内部地址。
- 自动导入只处理文件、元数据和基础标题，不会替你编造报错、根因或验证结果。
- 正式上线只提交当前文章；如果线上代码比本机更新，或本机存在其他未上线的代码提交，工具会停止并提示先同步，避免覆盖内容。
- 三类参考模板位于 `content\templates`。
